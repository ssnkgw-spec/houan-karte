# 一次資料の収集（Phase A）

「法案カルテ」の新規カルテを書くため、**一次情報だけ**を WebSearch / WebFetch で集め、`scripts/draft-karte.ts` に渡す素材 markdown を作る。報道・解説記事・ブログ・SNS・政党サイト（立場のある発信）は**素材にしない**。

## 入力
法案名・bill-id・論点のいずれか（例：「防災庁設置法案」「bousai」）。不明確なら国会の議案情報で正式件名を先に確定する。

## 許可ドメイン（これ以外は素材にしない）
`lib/primary-sources.ts` が唯一の真実。判定は同ファイルの `isAllowedSourceUrl()` の定義に従う：

- `*.go.jp` … 政府・各府省・内閣法制局・e-Gov・国会会議録検索・国立国会図書館（NDL）を包含
- `*.lg.jp` … 地方自治体
- `github.com/smartnews-smri` 配下 … スマートニュース 国会議案DB

許可外のURLを見つけても**素材に入れない**。除外したものは末尾に「除外したソースと理由」として一覧化する（報道・政党サイト等）。

## 収集の優先順
1. **議案情報・経過**：衆議院・参議院の議案ページ（提出日・付託委員会・審議経過・採決）
2. **法律案の本体**：要綱／法律案・理由／新旧対照表（提出府省 or 衆参のページ）
3. **会議録**：国会会議録検索システム（kokkai.ndl.go.jp）の委員会・本会議の逐語
4. **調査資料**：衆参の調査局・調査室の ISSUE BRIEF／立法と調査 等
5. **現行法**：e-Gov 法令検索（laws.e-gov.go.jp）で改正前の条文
6. **所管府省の資料**：審議会答申・パブコメ（public-comment.e-gov.go.jp）・説明資料

## 厳守事項（カルテの中立性を守る）
- **逐語引用を必ず採る**：各事実に、出典本文からの逐語スニペット（30字程度）を併記する。後段の draft-karte が claim-citation 束縛に使う。スニペットは本番サイトにも表示される（出典透明性）。
- **会議録は会議単位の直接 URL を必ず取得する**：`https://kokkai.ndl.go.jp/txt/<record_id>/0` 形式で、その発言が収録されている会議録の直接リンクを取得してから引用する。`https://kokkai.ndl.go.jp/` トップ・検索画面・`#/` 付き URL は sources に入れない。取得できない場合は「会議録 URL 未取得・要補完」と明記。
  - NDL 会議録の URL 取得方法: `https://kokkai.ndl.go.jp/` で「委員会名 + 日付」または「法案名」で検索 → 該当回の「本文」リンクを取得
  - 例: `https://kokkai.ndl.go.jp/txt/122105367X00620260512/0`（第221回・特別委・第6号・2026/5/12）
- **推測・一般知識で補完しない**：一次資料に書かれていないことは書かない。確認できない点は「未確認」と明記。
- **賛否・立場は発信元ラベルつき**：「どの会派・主体が、どの会議録で述べたか」を逐語で。運営の地の文に推奨・誘導表現を入れない。
- **URLは取得時点で生きているものだけ**。リンク切れは記録しない。

## 出力
`素材-<bill-id>.md`（git 管理外）を作る。8セクションに割り付けた構成：

```markdown
# 素材: <正式件名>（<bill-id>）
収集日: YYYY-MM-DD

## 出典一覧（sources）
- [1] <タイトル> — <URL>
- [2] ...

## s1 前提となる現行法・現行体制
- 事実: …（出典 {1} / 逐語: "…30字程度…"）

## s2 何が課題とされているか（L3・発信元ラベル必須）
- <会派/主体>（{2} / 逐語: "…"）: …

## s3 こう変わる（現行→改正後）
## s4 対案・主な論点（L3）
## s5 会期と採決
## s6 これまでの経緯
## s7 設計・方向性の選択（L3）
## s8 よくある声と射程（L3）

## 除外したソースと理由
- <URL>: 報道のため一次情報の対象外 など
```

## 次のステップ（案内まで行う）
1. `node --import tsx scripts/draft-karte.ts <bill-id> 素材-<bill-id>.md`（claude -p で下書き＋検証レポート生成・APIキー不要）
2. **人間ゲート §6-3**：`review-report-<bill-id>.md` 末尾の4項目チェックを人が通す
3. `content/bills/<bill-id>.ts` として確定 → `content/bills/index.ts` に登録 → `gianTitle` に正式議案件名（完全一致）を設定 → `npm run build`
