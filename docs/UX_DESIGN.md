## 1. ジョブストーリー

### P1: 関心ある一般市民（主ペルソナ）
報道の論調にも陰謀論にも与したくない。一次情報で自分で判断したいが、官庁サイトは難しくて続かない。

- **J1（最重要）**: SNS/検索で話題の法案名を見かけたとき、その法案で**何が・どう変わるのか**を中立に知りたい。だから報道のフレーミングに流されず自分で判断できる。
- **J2**: 選挙や時事の話題に触れたとき、**いま国会で何が動いているか**の全体像をざっと把握したい。だから話題の文脈がつかめる。
- **J3**: ある記述に「本当か？」と疑問を持ったとき、**根拠の一次資料に即座に当たりたい**。だから他人の要約を鵜呑みにせず確かめられる。

> 補強（実在シグナル）: Z世代の7割超が投票意向ありだが「政党の政策や候補者の違いがよくわからない」が低投票率の背景（SHIBUYA109調査2023）。「関心はあるが判断材料がない」層がJ1/J2の中核。

### P2: 学生・教育現場（副ペルソナ）
- **J4**: 授業で法案を扱うとき、**1つの法案を構造的に（現行法→課題→改正点→論点）**見せたい。だから主権者教育の生きた教材にできる。

### P3: 記者・ライター・調べ物層（副ペルソナ）
- **J5**: ある法案を取材・執筆するとき、**一次資料の所在と審議経過を素早く一覧したい**。だから裏取りの起点にできる。

## 2. 主要動線（critical path）

### 動線A（最重要）: カルテ直接着地 → 理解 → 一次資料で確認 ［J1・J3］

現実のトラフィックの大半は「トップ」でなく「個別カルテ」に外部リンクから着地する。ここが成立しないとサービスは機能しない。

```
[画面: 法案カルテ冒頭]
  registry card（法案名・種別・審議状況・一次資料リンク）＋「このページの方針」中立性ノート＋TOC
  → First Action: TOC で関心セクション（多くは ③こう変わる / ④論点）へジャンプ
[画面: ③ 現行↔改正後の対比]
  「今（現行）」→「改正後」を左右で対比、影響（権限/規模/法改正）を3点
  → First Action: 変更点を理解し、気になる記述の出典[n]を押す
[画面: 出典（一次資料）/ 一次資料へ遷移]
  → ユーザーが原典で裏を取る、または別セクションへ
[ゴール: 報道の論調を介さず「何が変わるか」を把握し、必要なら一次資料に到達できた]
```

**この動線の成立条件**: 着地して最初の1画面で「中立スタンス（結論を出さない・全部に出典）」と「TOCでどこに何があるか」が同時に伝わること。

### 動線B: トップで全体把握 → 個別カルテ ［J2・J4・J5］

```
[画面: トップ＝今国会ダッシュボード]
  会期クロック / 成立件数 / 議席構成 / 読む前の前提 / 審議中カード一覧
  → First Action:「いま審議されている法案」カード群から関心ある1枚を選ぶ
[画面: 法案カルテ]
  → 動線A の流れに合流
[ゴール: 全体の文脈の中で個別法案を理解できた]
```

## 3. 評価基準（数えられる形）

着地後の「使える感」を主観でなく検証する。

| # | 基準 | 目標値 |
|---|---|---|
| E1 | カルテ着地から「この法案で何が変わるか」に到達 | 1〜2スクロール / 30秒以内 |
| E2 | 中立スタンス（結論を出さない・全記述に出典）が伝わる | 着地後10秒以内 |
| E3 | 任意の事実記述から一次資料へ到達 | 1クリック |
| E4 | トップから任意のカルテへ到達 | 1クリック |
| E5 | カルテ内で「次にどこを読めばいいか」迷う場面 | 0回（TOC常時表示） |
| E6 | 「参加の経路」が押し付け・呼びかけと誤解される | 0（任意であることが明示） |
| E7 | スマホ幅で対比表（③）が破綻せず読める | 縦積みで成立 |

## 4. 既存プロトタイプの動線評価（読解ベース）

新規にArtifactsで作り直す必要はない（高品質な実物が存在）。既存7枚を上記基準で評価した。

| 基準 | 判定 | 所見 |
|---|---|---|
| E1 | ✅ | カルテに sticky TOC があり ③へ1タップ。③は old↔new 対比＋影響3点で「何が変わるか」が明快。 |
| E2 | ✅ | 冒頭 registry card 直下に「このページの方針」中立ノート。原則タグ（一次情報だけ/結論は出さない/出典リンク）も明示。 |
| E3 | ✅ | 上付き出典 `[n]` が REF スクリプトで一次資料へ直リンク。末尾に出典一覧。 |
| E4 | ✅ | トップの審議中カードが各カルテへ直リンク（`a.billcard`）。 |
| E5 | ⚠️ | sticky TOC は強力。ただしカルテ間（例: bousai→トップ）の戻り導線は末尾テキストリンクのみ。**全ページ共通のヘッダー（トップへ/法案一覧へ）が無い**のが唯一の弱点。 |
| E6 | ✅ | 参加セクション・sanka.html とも「呼びかけではない/伝えないことも正当」を明示。設計の白眉。 |
| E7 | ✅ | `@media(max-width:620px)` で ③対比は縦積み＋矢印回転に対応済み。 |

### 改善が要るのは1点だけ（実装時に対応）
- **全ページ共通のグローバルヘッダー（サイト名＝トップへ / 法案一覧）を追加**。現状トップへの戻りがフッター付近のテキストリンク頼みで、カルテ直接着地（動線A）したユーザーが「他の法案も見る」に移りにくい。Next.js移植時に共通レイアウトで解消できる。
